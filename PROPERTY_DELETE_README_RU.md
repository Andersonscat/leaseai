# 🗑️ Удаление недвижимости - Быстрый старт

## Что реализовано

✅ **Безопасное удаление** - объекты помечаются как удаленные, но остаются в БД  
✅ **API эндпоинт** - DELETE `/api/properties/[id]`  
✅ **UI кнопка** - красная кнопка "Delete Property" на странице объекта  
✅ **Модальное окно** - подтверждение перед удалением  
✅ **Уведомление** - toast сообщение об успехе  
✅ **Фильтрация** - удаленные объекты не отображаются в списках  

## Как использовать

### 1. Применить миграцию к базе данных

Откройте Supabase Dashboard → SQL Editor и выполните:

```sql
-- Содержимое файла: supabase/add-soft-delete-properties.sql

ALTER TABLE properties 
ADD COLUMN IF NOT EXISTS deleted_at TIMESTAMPTZ DEFAULT NULL;

CREATE INDEX IF NOT EXISTS idx_properties_deleted_at ON properties(deleted_at);

COMMENT ON COLUMN properties.deleted_at IS 'Timestamp when property was soft deleted. NULL means active.';
```

### 2. Готово! Используйте приложение

1. Откройте страницу любого объекта недвижимости
2. Нажмите красную кнопку **"Delete Property"** в правом верхнем углу
3. Подтвердите удаление в модальном окне
4. Вы будете перенаправлены на dashboard с уведомлением об успехе

## Архитектура решения

### Soft Delete (Мягкое удаление)

Вместо полного удаления из БД, мы устанавливаем поле `deleted_at`:

```
deleted_at = NULL      → объект активен
deleted_at = timestamp → объект удален
```

**Преимущества:**
- ✅ Можно восстановить
- ✅ Сохраняются связи с контрактами и арендаторами
- ✅ История не теряется
- ✅ Безопаснее для данных

### Безопасность

- ✅ Только владелец может удалить свою недвижимость
- ✅ Проверка на уровне API
- ✅ Модальное окно предотвращает случайное удаление
